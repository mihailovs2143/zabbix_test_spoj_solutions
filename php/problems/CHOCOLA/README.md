# CHOCOLA - Chocolate Breaking

## Описание
Решение задачи SPOJ CHOCOLA - разбить шоколадку m×n на квадратики 1×1 с минимальной стоимостью.

## Алгоритм: Greedy (Жадный)

### Ключевая идея:
**Стоимость разреза умножается на количество существующих кусков в перпендикулярном направлении!**

### Стратегия:
1. Сортируем все разрезы (вертикальные и горизонтальные) по убыванию стоимости
2. Жадно выбираем самый дорогой разрез
3. Умножаем стоимость на текущее количество кусков
4. Обновляем счётчик кусков

### Почему это работает?
- Дорогие разрезы выгоднее делать **раньше** (когда кусков мало)
- Дешёвые разрезы можно делать **позже** (даже если придётся повторять для многих кусков)

## Сложность
- **Время**: O((m+n) log(m+n)) - сортировка разрезов
- **Память**: O(m+n) - хранение массивов разрезов

## Пример работы

### Входные данные:
```
6×4 шоколадка
Вертикальные разрезы (5): [2, 1, 3, 1, 4]
Горизонтальные разрезы (3): [4, 1, 2]
```

### Решение:
```
Сортируем по убыванию:
vertical: [4, 3, 2, 1, 1]
horizontal: [4, 2, 1]

Жадный выбор:
1. v(4) vs h(4) → выбираем v(4): 4*1 = 4, vPieces=2
2. v(3) vs h(4) → выбираем h(4): 4*2 = 8, hPieces=2
3. v(3) vs h(2) → выбираем v(3): 3*2 = 6, vPieces=3
4. v(2) vs h(2) → выбираем v(2): 2*2 = 4, vPieces=4
5. v(1) vs h(2) → выбираем h(2): 2*4 = 8, hPieces=3
6. v(1) vs h(1) → выбираем v(1): 1*3 = 3, vPieces=5
7. h(1) остался: 1*5 = 5, vPieces=6
8. v(1) остался: 1*4 = 4, hPieces=4

Итого: 4+8+6+4+8+3+5+4 = 42 ✅
```

## Формат входных данных

```
<количество тестов>
<пустая строка>
<m> <n>          ← размер (m столбцов, n строк)
<x1>             ← стоимости вертикальных разрезов
<x2>             ← (всего m-1 штук)
...
<x(m-1)>
<y1>             ← стоимости горизонтальных разрезов
<y2>             ← (всего n-1 штук)
...
<y(n-1)>
<пустая строка>  ← между тестами
```
